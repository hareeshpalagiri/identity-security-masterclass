📘 01 — Windows Event IDs (Security Monitoring Reference)
A medium‑depth, Mahabharata‑aligned guide to the most important Windows Security Event IDs for Monitoring, Logging & Incident Response.

🏹 **Mahabharata Analogy —
“Reading the battlefield signals before danger strikes.”**
In the Mahabharata:

🔐 1. What Are Windows Event IDs? (Simple Definition)
Windows Event IDs are numerical codes generated by the Windows Event Log system.
They record:

logons
logoffs
authentication failures
privilege use
process creation
directory service changes
policy modifications
attacker behaviors

Understanding them is essential for:

Threat hunting
SOC triage
Investigation
Incident response
Detection engineering


🧠 2. Most Important Security Event IDs (Medium-Depth)
Below are the most critical Windows Event IDs grouped by category.

⚔️ 3. Logon / Authentication Event IDs
Event ID 4624 — Successful Logon
Indicates successful authentication.
Useful for:

Lateral movement detection
Privileged account misuse
Logon type analysis

Key logon types:

3 → Network
10 → Remote desktop
2 → Interactive
11 → Cached interactive


Event ID 4625 — Failed Logon (Brute Force / Spray)
Triggers when authentication fails.
Check for:

Password spray attempts
RDP brute force
Account enumeration


Event ID 4648 — Logon Using Explicit Credentials
Signals credential theft attempts:

Pass‑the‑Hash
Admin elevation
Privilege misuse


Event ID 4768 — Kerberos TGT Request
Triggered when user requests a Kerberos TGT (AS‑REQ).
Useful for:

Tracking authentication types
Early-stage attacks
AS‑REP roasting detection


Event ID 4769 — Kerberos Service Ticket Request (Silver Ticket / Kerberoasting)
Occurs when a TGS ticket is requested.
Key for:

Kerberoasting
Service account abuse
Ticket replay detection


🔎 4. Account Management Event IDs
Event ID 4720 — User Account Created
Monitor in domain environments for:

Rogue admin creation
Unauthorized provisioning


Event ID 4722 — Account Enabled
Watch for attackers re‑enabling disabled accounts.

Event ID 4723 & 4724 — Password Reset / Change Attempt
Useful for detecting:

Compromised admin resetting user passwords
Illicit password changes


Event ID 4732 / 4728 — User Added to Privileged Group
Critical events for:

Domain Admins
Enterprise Admins
Server Operators
Backup Operators

If suspicious → possible privilege escalation.

🧱 5. Object Access Event IDs
Event ID 4663 — File / Folder Access
Used for:

Data exfiltration detection
Sensitive folder monitoring


Event ID 4656 — Handle Requested for Object
Precedes 4663 — early indicator of interest in sensitive files.

🖥 6. Process / Execution Event IDs
Event ID 4688 — New Process Created
One of the MOST important events.
Useful for:

Malware execution
PowerShell attacks
LOLBins (Living-off-the-Land binaries)
Suspicious command lines
Mimikatz detection


Event ID 4697 — Service Installed
Attackers often install:

malicious services
persistence mechanisms


Event ID 7045 — New Service Installed (System Event)
Similar to 4697 but in System Log.
Useful for IR → persistence detection.

🧩 7. Policy, GPO, and Security Events
Event ID 4739 — Domain Policy Changed
Critical for:

Attackers modifying domain security
Weakening password policies


Event ID 4719 — System Audit Policy Modified
Attackers often disable auditing before performing malicious actions.

Event ID 4735 / 4737 — Security Group Modified
Incorrect modification → privilege escalation.

🧬 8. Directory Service Events (For Domain Controllers)
Event ID 4662 — Directory Service Access
Monitors:

DCSync
AD object access
Privileged attribute changes


Event ID 5136 — Directory Object Modified
Useful for detecting:

RBCD attacks
AdminSDHolder modification
ACL changes


Event ID 1102 — Audit Log Cleared
One of the most critical IR alerts.
If unexpected → possible attacker cleanup.
